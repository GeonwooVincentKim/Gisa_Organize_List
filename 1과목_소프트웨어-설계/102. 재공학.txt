II. 재공학
1. 재공학
- 1. S/W Reengineering 의 개념 및 목표
  = 1. S/W 위기 개발 생산성 X, 유지보수 생산성 O
  = 2. 재구조화 -> 재공학 한 유형, 사용자 요구사항, 기술적 설계 변경 없이 프로그램 개선
  = 3. S/W 재공학, 예방 유지보수 (Preventive Maintenance)
  = 4. 재사용 수월, S/W 수명 연장

- 2. 재공학 장점
  = 1. 개발 시간 비용 & 프로젝트 실패 위험 감소
  = 2. S/W 품질 및 생산성 향상 & 구축 방법 개발 지식 공유

- 3. 재공학 과정
  = 1. 분석 (Analysis) -> S/W 명세서 확인, S/W 동작 이해, 재공학 대상 선정
  = 2. 재구성 (Restructuring) -> S/W 구조 향상, 코드 재구성
  = 3. 역공학 (Reverse Engineering) -> 원시 코드 분석, S/W 관계 파악, 기존 시스템 설계 정보 재발견, 다시 제작
  = 4. 이식 (Migration) -> 기존 S/W System 을 새로운 기술, H/W 환경에서 사용할 수 있도록 변환하는 작업

- 4. 재사용 2가지 기본 기술
  = 1. 생성 중심 (Generation Based, 모듈화) -> 재사용 단위 찾아 발전, 유용한 S/W 부품 찾아내기
  = 2. 합성 중심 (Composition Based, 모델화) -> 모듈을 생산성 있게 조립, 모듈 만들어서 끼워 맞추는 방법, S/W 완성

- 5. 리팩토링 (Refactoring) -> 쉽게 이해, 적은 비용 수정, 겉으로 보이는 동작 변화 없이 내부 구조 변경

** 3초 컷 -> 기존 S/W System 을 새로운 기술, H/W 환경에서 사용할 수 있도록 변환하는 작업 -> Migration (이식)

2. CASE (Computer Aided S/W Engineering)
- 1. CASE
  = 1. 요구 분석, 설계, 구현, 검사, 디버깅 -> 전용 S/W 도구
  = 2. 자료 흐름도 (다이어그램), 작업 과정 & 데이터 공유 -> 작업자 간 커뮤니케이션 증대

- 2. CASE 가 제공하는 기능
  = 1. 신속 개발, 오류 수정 용이, 품질 향상
  = 2. S/W 생명주기 전체 단계 연결, 자동화, 통합 도구 제공
  = 3. S/W 시스템 문서화, 명세화 (Graphic), 개발 단계 표준화, 자료 흐름도 작성 기능
  = 4. 모델들, 모순 검사 기능 제공, S/W 개발 모형 지원
  = 5. 원천 기술 : 구조적 기법, 프로토타이핑 & 정보 저장소 기술

- 3. CASE 사용의 장점
  = 1. S/W 개발 기간 단축 & 비용 절감 -> S/W 생산성 향상
  = 2. 자동화된 검사, S/W 품질 향상
  = 3. 프로그램 유지보수 간편, S/W 모듈의 재사용성 향상
  = 4. S/W 개발 주기 표준안 확립, S/W 개발 기법 실용화, 문서화 용이성 제공, System 수정 & 유지보수 축소 효과

- 4. CASE 의 분류
  = 1. 상위 (Upper) Case -> 요구 분석, 설계단계 지원 (Model 간 모순 검사 기능, 오류 검증, 자료 흐름도 작성)
  = 2. 하위 (Lower) Case -> 소스 코드 작성, Test, 문서화 과정 지원
  = 3. 통합 (Integrate) Case -> S/W 개발 주기 전체 과정 지원

** 3초 컷
1. CASE 의 원천 기술 -> 구조적 기법, 프로토타이핑 기술, 정보 저장소 기술
2. CASE 의 분류 -> 상위, 하위, 통합
**

3. 요구사항 분석을 위한 CASE 도구
- 1. 요구사항 분석을 위한 CASE
  = 1. 요구사항 자동 분석, 분석 명세서 기술, 개발된 도구
  = 2. 표준화, 보고 -> 문서화 품질 개선, 추적 용이성, 유지보수 비용 축소, 결함 & 생략 & 불일치 (발견 용이성)

- 2. 요구사항 분석을 위한 CASE 도구
  = 1. SADT (Structured Analysis and Design Technique)
   -> SoftTech 사, 시스템 정의, S/W 요구사항 분석, 시스템/ SW 설계, 구조적 분석 & 설계 도구 + 블록 다이어그램

  = 2. SREM (S/W Requirements Engineering Methodology)
  -> 1. REL/REVS : TRW 사, 우주국방시스템 그룹, 실시간 처리 S/W 시스템, 요구사항 명확 기술, RSL + REVS 자동화
  -> 2. RSL (Requirement Statement Language) => 요소, 속성, 관계, 구조들 기술, 요구사항 기술 언어
  -> 3. REVS (Requirement Engineering and Validation System) 
     => RSL, 기술 요구사항, 자동 분석 - 요구사항 분석 명세서 출력하는 요구사항 분석기
    
  = 3. PSL/PSA -> 미시간 대학, PSL & PSA 사용하는 자동화 도구

** 3초 컷
1. 요구사항 분석을 위한 CASE 도구 -> SADT, SREM, PSL/PSA
2. CASE 도구는 모두 약어에 S 가 포함
